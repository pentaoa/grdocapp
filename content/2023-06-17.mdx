本网站的 0.2 版本本人总体上是比较满意的。它删除了大量插件，大幅减少了页面元素，使用了轻量级设计，我们肯定繁复的价值，但是化繁就简总令人豁然开朗。我将从四个方面来说说本网站的设计。

### 技术路线

这个网站虽然看起来非常简陋，但是实际上使用了可以说世界上最先进的网站框架——Next.js 13 （目前无中文技术支持和文档），最先进的 Serverless——Vercel ，最有前景的 CSS——Tailwind CSS 。（但是，显然缺少一个能驾驭他们的脑子）

### 用户界面设计

苹果公司在21年对其 iOS 16 上的 Safari 浏览器的交互进行了重新设计，把标签栏往下面移动，并称其让浏览更顺手，此外这种设计其实对移动端的整个浏览体验有着极大提升：网站本身的内容、导航栏等元素不会随着上划下划而上下跳动，导致视觉错乱。

> Safari introduces a newly designed browsing experience that makes controls easier to reach. By default, the new tab bar is positioned at the bottom of the screen, so users can easily swipe between tabs with one hand. Tab Groups allow users to organize tabs and easily access them at any time across iPhone, iPad, or Mac, and a customizable start page and web extensions on iOS make Safari more powerful and personal than ever.

![Apple_iOS15-avail_safari_09202021](https://ihs-1317591693.cos.ap-guangzhou.myqcloud.com/grdoc/Apple_iOS15-avail_safari_09202021.jpg)

然后我就突发奇想，咱把 footer 改成导航栏，这样在桌面端就有一种打字机的感觉，让人们感受文字的重量；正在移动端更能提升交互质量，和国内其它网站拉开差距。

于是，这样一个 footer bar 就诞生了！

这个 bar 的长度令人感到非常舒适，总是与屏幕边缘保持一定的间距。在大屏幕中，这个 bar 会同时显示图标和其他小组件，在小屏幕中，会只留下最重要导航功能。以后技术成熟了再给它加点动画，不就成灵动岛了吗？

bar 上的滑块设计灵感来源于 Lee Robinson （Vercel 的开发者体验副总裁）的 Blog ，滑块点起来真的很有快感！

![bar](https://ihs-1317591693.cos.ap-guangzhou.myqcloud.com/grdoc/bar.png)

此外，给大部分元素加上 “shadow-xl rounded-lg” 的类名，即阴影和圆角设计，这是最简单的让设计有质感和纹理的方法。

### 色彩和动画

为什么要把网站的背景色弄成白色的？因为白色是最适合展示给别人看的颜色（反之，黑色系配色经常在生产力软件中看到，因为他们能扩展创造力）。此外，我喜欢白色，白色就是所有颜色。

但是，网站不全使用单纯的白色，footer bar 使用了一种名为 "stone-300" 的白色，使用它是为了和背景区分开，让网站更有层次感。这个白色有点像石灰，也有点像牛奶，总之，我很喜欢（有一段时间比较喜欢乳白色，后面就一般般了）。这种白色的 rgb 为 "214 211 209"。然后，还对那个 bar 加入了高斯模糊（高斯模糊本身比较复杂，但是 Tailwind CSS 让使用它变得非常简单，想要使用请移步其官网）（不是所有浏览器都支持模糊效果）。

"Framer Motion" 这个玩意可以让我们在网页里添加物理效果动画。比如这个 Footer Bar 上的滑块，是不是很 Q弹？为了设计这样的效果，需要将其 transition 设置为

```typescript
type: 'spring',
stiffness: 444,
damping: 20,
```

（让白色框框能随意移动，花了不少功夫）。

### 可访问性❤️

- 不滥用光影。所有的阴影都是在有明显边界处使用，不使用对视觉障碍者不友好的新拟物设计。
- 所有主要可交互元素均可通过键盘进行聚焦与交互。
- 元素语意化处理（本网站还未完成）。

语意化是啥？

> 语义化，指对文本内容的结构化（内容语义化），选择合乎语义的标签（代码语义化），便于开发者阅读，维护和写出更优雅的代码的同时，让浏览器的爬虫和辅助技术更好的解析。

语意化可以使网站的源代码更易读，更好适配浏览器的阅读器。

### 排版的小问题

#### 响应式设计

在以前，编写网页用插件，用模板，但是这个网站是自己构建的，才让我意识到，响应式是那么重要，也那么令人头疼。

> 早年设计 Web 时，页面是以适配特定的屏幕大小为考量创建的。如果用户正在使用比设计者考虑到的更小或者更大的屏幕，那么结果从多余的滚动条，到过长的行和没有被合理利用的空间，不一而足。随着人们使用的屏幕尺寸的种类越来越多，出现了响应式网页设计的概念（*responsive web design，RWD*），RWD 指的是允许 Web 页面适应不同屏幕宽度因素等，进行布局和外观的调整的一系列实践。这是改变我们设计多设备网页的方式的思想，在这篇文章里，我们将会帮你理解掌握它时所需知道的主要技能。

[响应式设计的说明](https://developer.mozilla.org/zh-CN/docs/Learn/CSS/CSS_layout/Responsive_Design)

#### 多语言

首先，有一个非常重要的东西。

1.中英文之间要加空格。

2.一个混合词组中不加空格，如：欢迎加入QQ群讨论

3.中文数字间加空格。

这是一个非常重要的排版规范。以至于 “Paranoid text spacing in JavaScript” 项目（GitHub 3.9k stars）在 Readme 中写到：

> 有研究显示，打字的时候不喜欢在中文和英文之间加空格的人，感情路都走得很辛苦，有七成的比例会在 34 岁的时候跟自己不爱的人结婚，而其余三成的人最后只能把遗产留给自己的猫。毕竟爱情跟书写都需要适时地留白。与大家共勉之。

所以，建议屏幕前的你在今后的正式书写中多加空格，不管是写论文还是发公众平台，都应注意这个规则。

但是现在有一些软件已经能自动帮你加入了，比如 iOS 系统，比如 Word （Word 的编辑体验是非常不错的）。这就引发了一些争议：

> Office 等软件支持自动调整中英文字符的间距，手动加空格的意义何在？
>
> 确实，手动加空格在 Word 中显得没有意义。
>
> 但考虑到个人大多数的编辑活动并不是在 Word 上进行，并且目前以及未来很长一段时间，很多编辑器，网页或者排版系统都没有自动调整中英字符间距的功能。为了自己的码字体验和网上大多数用户的阅读体验，手动添加空格是一个可以被接受的做法。
>
> 有没有考虑过未来的排版系统进化后，全部实现了自动空格，旧时代的「空格党」们该何去何从？
> 自动化的解决方案毫无疑问是一件好事，也是未来的趋势。无论对空格党还是非空格党都是如此。
>
> 现阶段的手动空格做法，无论被认为是 Dirty Hack 也好，Workaround 也罢，都是人们积极寻求当前解决方案的一种尝试和实践，所以手动空格某种程度上甚至可认为是一种基于人文的「自动化」解决方案。
>
> 而未来排版渲染的改善，如果能够兼容旧的解决方案，或者将旧的格式转换为新的格式的话（比如你在 V2EX 提交评论时无论是否加了手动空格，最终都被强制重排版为统一的格式），对于「空格党」和「非空格党」们来说都是可以接受的。



总之，简单的外表下用于规范设计的代码真不少，所有这一切，只为你的浏览体验！
